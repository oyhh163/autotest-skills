# 测试体系流程模板

## 输出目录

为每次运行创建带时间戳的目录：

```text
test-output/YYYYMMDD-HHMM-功能名称/
```

推荐内容：

```text
requirements-analysis.md
test-cases.xlsx
test-cases.xmind
automation-cases.md
automation/
automation-results/
questions-risks.md
test-knowledge.md
```

将缺陷汇总工作簿保存到用户桌面：

```text
Desktop/automation-defects-YYYYMMDD-HHMM.xlsx
```

## 自动化模块

当项目没有自动化标准时，在运行输出或项目测试区域创建以下结构：

```text
automation/
  README.md
  cases/
  fixtures/
  reports/
  screenshots/
```

`README.md` 必须包含：

```markdown
# UI自动化

## 范围
- 来源用例：[用例ID]
- 目标端：[Web/App]

## 所需输入
- URL或包名/Bundle ID：
- 平台/浏览器：
- 测试账号：
- 测试数据：
- 设备/模拟器：

## 运行命令
[命令]

## 备注
- [已知阻塞点]
```

### Web自动化默认设置

除非项目已使用其他UI自动化框架，否则使用Playwright。

询问：

```text
Web URL
登录方式
测试账号
目标浏览器
环境
是否允许mock
```

### App自动化默认设置

除非项目已使用其他移动自动化框架，否则使用Appium。

询问：

```text
包名或Bundle ID
Android或iOS
设备/模拟器名称
如需要提供应用安装路径
测试账号
环境
```

## 缺陷Excel

在桌面创建 `automation-defects-YYYYMMDD-HHMM.xlsx`。

列：

```text
Bug ID
来源用例ID
自动化用例ID
目标端
模块
功能
严重程度
标题
实际结果
预期结果
复现步骤
环境
证据路径
可疑原因
状态
需求歧义
```

严重程度指南：

```text
阻塞（Blocker）：功能不可用、数据丢失、支付/安全问题，或发布无法继续
严重（Critical）：核心路径中断或严重用户影响且无变通方案
重要（Major）：重要功能异常但存在变通方案
次要（Minor）：边界情况、文案、样式或低影响问题
```
